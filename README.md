# 月結び Public Beta

ふたりが毎月、関係を続けたいか、今月どう感じたかを密封して提出する LINE MINI App 向け Web アプリです。

このディレクトリは公開版専用です。既存の個人版とはコード・データベース・LINEチャネルを分離してください。

## 実装済み

- 任意の呼び名・一文字アイコン・記念日
- 作成者用復元コードとパートナー招待コード
- 毎月1回、6項目を1〜10で採点し、4つの更新意思から選択
- 双方の提出前は回答を相互非公開
- 完成した契約の履歴、詳細、総合得点と推移
- 記念日、デート記録、ふたりで行きたい場所
- 無料版24枚／Plus 300枚の非公開共有アルバム（JPEG、1枚900KB以下に自動圧縮）
- Plus向けテーマ質問、得点差・前月比・会話テーマの関係レポート
- ペア単位のPlus権限、有効期限、IAP注文を保持するサーバー側データ構造
- 日中二言語
- データ書き出し、プロフィール更新、アカウント削除
- 1か月290円／3か月790円の買い切りパス表示（自動更新なし）

## Free / Plus

無料版は月次契約、基本履歴・得点推移、記念日3件、デート10件、行きたい場所10件、写真24枚を利用できます。Plusはテーマ質問と関係レポートを解放し、各保存上限を記念日100件、デート500件、行きたい場所200件、写真300枚へ拡張します。一度の購入で同じペアの2人に適用します。

現時点ではLINE課金審査前のため、購入ボタンは無効です。クライアント側だけでPlusを有効化する処理や、偽の購入完了処理は実装していません。

## ローカル確認

静的サーバーでこのフォルダを公開してください。`config.js` が空の場合は、このブラウザだけに保存するデモモードで起動します。設定画面からA/Bの役割を切り替えると、双方提出の流れを1台で確認できます。

## 新しい Supabase を接続

1. 公開版専用の新規 Supabase project を作成します。
2. Authentication → Sign In / Providers で Anonymous sign-ins を有効にします。
3. SQL Editor で `supabase/schema.sql` を実行します。
4. 既存環境へのPlus追加は `supabase/migrations/20260714_plus_membership.sql` を1回実行します。新規環境でもschemaの後に同じmigrationを実行します。
5. `config.example.js` を参考に `config.js` に Project URL と Publishable key を設定します。
6. Storage の `couple-album` が Private になっていることを確認します。

Publishable key はブラウザに置いてよい公開キーです。`service_role` key は絶対に配置しないでください。

## LINE MINI App

公開版専用の LINE MINI App channel を作成し、開発用 LIFF ID を `config.js` に設定します。Endpoint URL は公開版の HTTPS URL にします。現在の個人版チャネルは変更しません。

## 公開前に必要

- 運営者名・連絡先を入れた正式な利用規約とプライバシーポリシー
- LINE ID token のサーバー検証と端末変更フローの最終化
- 通報・問い合わせ窓口、障害監視、バックアップ
- Cloudflare Pages 等、商用サービスを許可するホスティングへの配置
- LINE の審査、課金を行う場合は入管確認後に IAP 申請

LINE IAPを有効化する際は、認証済みMINI App、IAP審査、サーバーでの購入検証・Webhook処理が必要です。検証済み注文だけを `iap_orders` に保存し、同じトランザクションで `pair_entitlements` を延長してください。ブラウザからこの2テーブルへ書き込む権限は与えていません。
